# Nox ERC20ToERC7984Wrapper — Decrypt Bug Reproduction

## The Bug

Any contract using `ERC20ToERC7984Wrapper` or `ERC7984._update()` (including code generated by [cdefi-wizard.iex.ec](https://cdefi-wizard.iex.ec/)) can **shield** tokens but **cannot decrypt** them. The Handle Gateway returns `404 Object not found` for all handles produced by `safeAdd`/`select` chains in `_update`.

Only `Nox.add()`/`Nox.sub()` (PiggyBank pattern from Hello World docs) produce decryptable handles.

## Environment

- Network: Arbitrum Sepolia (421614)
- `@iexec-nox/nox-protocol-contracts@0.1.0`
- `@iexec-nox/nox-confidential-contracts@0.1.0`
- `@iexec-nox/handle@0.1.0-beta.10`
- Compiler: `solc 0.8.34`, `evmVersion: paris`

## Reproduce

```bash
npm install
npx hardhat compile
node test/reproduce.js
```

## Expected

Both wrappers should decrypt the balance.

## Actual

```
Wizard Wrapper (ERC20ToERC7984Wrapper):  ❌ 404 Object not found
PiggyBank Wrapper (Nox.add/sub):         ✅ 500.0 — instant decrypt
```

## Root Cause

`ERC7984Base._update()` uses `Nox.safeAdd()` + `Nox.select()` which create chained computation handles. The Nox Runner does not process these — the ciphertext is never stored in the Handle Gateway.

`Nox.add()` / `Nox.sub()` produce handles the Runner processes instantly.

## Contracts

- `contracts/WizardWrapper.sol` — exact code from cdefi-wizard.iex.ec (broken)
- `contracts/PiggyWrapper.sol` — same functionality using Nox.add/sub (works)
- `contracts/MockERC20.sol` — simple test token
